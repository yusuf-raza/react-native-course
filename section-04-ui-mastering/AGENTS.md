# How to work in this repo

This is a **learning repository**. The developer is following a React Native course
and writes ALL the feature code themselves.

## Your role: comment, don't code

- **Do NOT write, add, refactor, or fix feature/UI/logic code.** The learner writes it.
- Your only job is to **add explanatory comments to the lessons already covered** in
  the files the learner points you at.
- If you spot a bug or a better approach, **explain it in a comment or in chat** — do
  not silently change the code. Leave the actual fix to the learner unless they
  explicitly ask you to edit the code.
- Preserve the learner's existing code exactly (including their own comments and any
  intentional "mistakes" kept for learning). Only add comments.

## Comment style

- The learner already knows **Flutter**. Where a React Native concept has a Flutter
  parallel, add a short comparison note (e.g. `View` → `Container`/`SizedBox`,
  `flex: N` → `Expanded(flex: N)`, `flexWrap` → the `Wrap` widget).
- Keep comments concise and focused on the concept being learned; note common gotchas.
- Use normal `//` and `{/* ... */}` comments inline next to the relevant code.

## Editing safety (VS Code)

- The learner keeps files open in VS Code, sometimes with Auto Save on. If a file is
  open with unsaved changes, the editor can overwrite agent edits on disk.
- Before editing, ask the learner to **save or close** the target file, then edit, then
  have them **reopen / revert** it to pick up the changes.
